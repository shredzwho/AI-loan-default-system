# AI-Loan Default Intelligence System - Technical Documentation

## Architecture Overview
The AI-Loan system has evolved from a basic machine learning pipeline into a comprehensive, enterprise-grade Next.js "Command Center" dashboard powered by a FastAPI Python backend. The system integrates traditional ML models (XGBoost) with advanced deterministic rule engines and Generative AI (Groq + Llama 3) to provide an end-to-end loan origination analysis platform.

## 1. Data Generation & Pipeline (`src/data/` & `src/features/`)
The foundation of the project relies on synthetic data generation and robust preprocessing.
*   **Data Generation (`load_data.py`):** Generates 1,000 highly realistic synthetic loan applicants. It goes beyond basic tabular data by injecting:
    *   **Time-Series Tracking:** 12-month arrays simulating checking account balance trajectories over time.
    *   **Behavioral Stress:** Injecting synthetic behavioral parameters for *Savings Velocity*, *Spending Volatility*, and *Utility Payment Lags*.
    *   **OSINT Fraud Indicators:** Synthetic document mismatch scores and identity fraud intent flags.
    *   **Graph Nodes:** Categorical markers representing employer industry risk (e.g., `Layoff_Alert`, `Market_Volatility`).
*   **Feature Engineering & Preprocessing (`build_features.py` & `preprocess.py`):** The pipeline handles missing values, engineers new features like the Debt-to-Income (DTI) ratio, label-encodes categorical variables (saving the encoders via `joblib`), and scales numerical features. 
*   **Data Imbalance Strategy:** The system utilizes **SMOTE** (Synthetic Minority Over-sampling Technique) to handle the inherently imbalanced default data. Crucially, SMOTE is applied *after* scaling to prevent data leakage.
*   **Model Training (`train.py`):** Supports Logistic Regression, Random Forest, and XGBoost. The default operational model is a tuned **XGBoost** model.

## 2. Core AI Intelligence Engines (`src/api/`)
The FastAPI backend (`api/app.py`) serves as the central orchestrator. When an analysis is requested, the borrower's data is pushed through 5 concurrent intelligence engines:

1.  **Explainability Layer (`explain.py`):** Calculates the core base probability of default (0% to 100%). It utilizes SHAP (SHapley Additive exPlanations) to extract the exact mathematical weights for every feature (e.g., "DTI Ratio increased risk by +12%").
2.  **Trust Graph Scorer (`trust_graph.py`):** Simulates a Graph Neural Network (GNN). It checks the borrower's connections (like their `Employer_Node`). If the employer's industry is flagged as high-risk, a cascading penalty (e.g., +15%) is applied to the borrower's base default risk.
3.  **Financial Stress Predictor (`stress_predictor.py`):** A deterministic anomaly detection algorithm that converts utility lags and spending volatility into a 0-100 Stress Score. It also computes a predictive "Risk Window" indicating when a default is likely to occur (e.g., "Critical: 1-3 Month Risk Window").
4.  **AI Loan Negotiator Engine (`negotiator.py`):** A recursive optimization function designed to save risky loans. If a borrower's default risk exceeds 40%, the engine loops through dropping the principal amount or extending the loan tenure until the borrower's projected Debt-to-Income (DTI) ratio falls below a safe 35% threshold.
5.  **Integrated Fraud Matrix (`fraud_detector.py`):** Calculates Fraud Intent scores vs. Default Ability. It categorizes borrowers into a 2x2 grid, outputting security classifications like "Sleeper Risk" (High Fraud Intent + High Ability to Pay) or "Criminal Fraud Ring".

## 3. Dual-Layer Explainable GenAI (`llm_explainer.py`)
To make raw ML coefficients digestible, the system interfaces with open-source Large Language Models via the **Groq API** (`meta-llama/llama-4-maverick-17b-128e-instruct`). It achieves two distinct goals asynchronously:
*   **GenAI Risk Narrative:** It translates abstract ML coefficients and SHAP values into a professional, objective 3-sentence executive summary for the loan officer.
*   **GenAI Recourse Health Coach:** It targets the borrower's specific negative risk factors and generates an immediate, highly actionable 3-step structured checklist representing a clear "Path to Approval".

## 4. Frontend Command Center (`frontend/src/`)
The user interface is a dense, high-performance Next.js and React dashboard styled purely with Tailwind CSS.
*   **Architecture:** The `page.tsx` dashboard serves as the master view, calling the FastAPI `/analyze_borrower` endpoint and consuming the massive, combined JSON payload generated by all 5 backend engines.
*   **Visualizations:**
    *   `SHAPChart.tsx`: A Recharts-powered bar chart displaying positive vs. negative feature drivers.
    *   `RiskMatrix2x2.tsx`: A glowing quadrant grid highlighting the OSINT threat profile and fraud intent.
    *   `StressTrajectoryChart.tsx`: A dynamic Recharts line-graph displaying the 12-month synthetic cash flow drain over time.
    *   `NegotiationPanel.tsx`: A multi-component analytical panel detailing alternative AI-generated loan paths.
*   **Design Tokens:** The application utilizes a premium deep-slate theme, featuring glassmorphism (`bg-slate-900/40`), neon accents, gradient maps, and Lucide telemetry icons.

## Adding New Features
To add a new AI analysis module:
1. Create a logical class in `src/api/` (similar to `stress_predictor.py`).
2. Instantiate the class in `api/app.py` within the `startup` block.
3. Execute the module in the `/analyze_borrower` endpoint and append the output to the master response payload.
4. Update the TypeScript `APIResponse` interface in `frontend/src/app/page.tsx`.
5. Create a new React component in `frontend/src/components/` and insert it into the Command Center grid.
